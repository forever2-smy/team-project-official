# AGENTS.md
# 项目智能协作规范 | 图形化 MUD 游戏
**适用**：AI 编程助手 / 新成员快速接入
**范式**：渐进式披露 | 最小必要信息

## 1. 项目架构概述
- 前后端分离：Node.js(Express) + Socket.IO + 浏览器前端
- 实时联机：WebSocket 双向同步玩家位置、聊天、状态
- 纯前端渲染：Canvas 2D 绘制地图与玩家
- 无数据库，内存状态，轻量可演示

## 2. 目录结构
```
/            项目根目录
├─ server.js  服务端：联机、玩家、地图、广播
├─ index.html 前端界面、样式、DOM
└─ game.js    客户端游戏逻辑、Socket、渲染
```

## 3. 核心模块职责
- server.js：玩家管理、移动校验、消息广播、地图生成
- index.html：UI 布局、输入框、按钮、画布容器
- game.js：连接服务端、键盘/按钮控制、Canvas 渲染、聊天

## 4. 编码规范约束
- 变量/函数：小驼峰 `playerMove`
- 类名：大驼峰 `MUDGame`
- 常量：全大写 `MAX_WIDTH`
- 事件命名：`player-join` `player-move`
- 函数单一职责，禁止超长函数
- 关键逻辑必须加注释

## 5. 禁止操作清单
- 禁止直接修改 `gameState`，必须通过方法更新
- 禁止硬编码 IP、端口、步长、颜色
- 禁止在渲染函数中写业务逻辑
- 禁止未校验直接修改玩家坐标
- 禁止破坏 Socket 事件名与通信协议
