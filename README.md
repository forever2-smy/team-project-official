## 3. 协作规范
### 3.1 分支命名规则
- 功能分支：feature/功能名-用户名（例：feature/订单模块-唐浩）
- 修复分支：bugfix/问题描述-用户名（例：bugfix/支付异常-唐浩）
- 文档分支：docs/文档名-用户名（例：docs/README-王嘉伟）

### 3.2 代码提交规范
```bash
# 提交格式：类型: 简洁描述（必填）
git commit -m "feat: 新增订单查询功能"  # 功能开发
git commit -m "fix: 修复支付接口参数错误" # Bug修复
git commit -m "docs: 更新README角色分配信息" # 文档更新
git commit -m "refactor: 重构订单模块代码结构" # 代码重构
